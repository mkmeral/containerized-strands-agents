The Problem

The Last 10% Problem

I built this for my own needs. Here's the background.

🛠️
Origin Story
When we had strands-command in devtools, the problem was always the last 10% — pulling repos, setting up environments, configuring tools. Everything before the agent could actually start working.
So I built containerized agents as a devtool. Agents run locally in Docker — I can see the files, change them, iterate. It's hands-on.
Why MCP (not A2A)? I wanted to share these agents with colleagues who use different devtools — Kiro, Claude Code, whatever. MCP is the universal protocol that works across all of them.
🧠
Context Bloat
50+ tools saturate the context window. Performance degrades, costs multiply.
⏳
No Parallelism
Sequential tool calls block. Research and coding can't run simultaneously.
🔓
No Isolation
All tools share one environment. A rogue command can destroy the workspace.
📦
Not Portable
Agent state is tied to the runtime. Can't move agents between environments.
The Solution

MCP: The Composability Layer

MCP isn't just a tool protocol — it's the infrastructure for composable agent systems.

🔧
Tools
Discoverable capabilities. Agents auto-detect what they can do. No hardcoded schemas.
📋
Tasks
Long-running operations with built-in status tracking, polling, cancellation, and TTL.
🔔
Notifications
Real-time status pushes. No manual polling — clients get updates as things happen.
📂
Resources
Agent workspaces become browsable content. Read what agents created, in the protocol.
💡
Prompts
Pre-built agent templates. Discoverable in any MCP client. Agent blueprints.
🔄
Composability
An MCP server can spawn agents that themselves use MCP servers. It's MCP all the way down.
Deep Dive

MCP Tasks: Async Agents,
Protocol-Native

Before: Fire-and-Forget

Python # Dispatch — no tracking result = send_message("researcher", "find issues") # → {"status": "dispatched"} # Manual polling loop... while True: msgs = get_messages("researcher") if msgs["processing"] == False: break sleep(5)
  • ✗ No status tracking
  • ✗ Manual polling required
  • ✗ No cancellation protocol
  • ✗ Context bloat from polling

After: MCP Tasks

Python # Dispatch → returns Task object result = send_message("researcher", "find issues") # → {taskId: "task-abc", status: "working"} # Protocol-native operations tasks/get(taskId) # → status: "completed" tasks/result(taskId) # → full output tasks/cancel(taskId) # → graceful stop tasks/list() # → all active tasks
  • ✓ Automatic status tracking
  • ✓ Built-in polling intervals
  • ✓ Native cancellation
  • ✓ Notifications on completion
Architecture

Containerized Strands Agents

An MCP server that spawns autonomous agents in Docker containers — each with their own MCP servers.

The Code

Minimal Code, Maximum Power

Agent Definition (~15 lines)

agent.py from strands import Agent from strands.tools.mcp import MCPClient from strands.session import FileSessionManager def create_agent(data_dir): # Auto-discover MCP tools from config mcp_tools = load_mcp_from_config( data_dir / ".agent/mcp.json" ) return Agent( system_prompt=load_prompt(data_dir), tools=[shell, editor, *mcp_tools], session_manager=FileSessionManager( session_dir=data_dir / ".agent/session" ), )

MCP Server (~20 lines)

server.py from fastmcp import FastMCP mcp = FastMCP("Agent Host") @mcp.tool async def send_message( agent_id: str, message: str, system_prompt: str = "default", ): """Dispatch a task to an agent.""" agent = await manager.get_or_create( agent_id, system_prompt ) task = await task_store.create_task( agent_id, message ) # Fire — status via tasks/get dispatch_async(agent, message, task) return {"taskId": task.id}

Portability

Same Agent, Any Runtime

Snapshot the agent's data directory. Restore it anywhere. Continue where you left off.

CLI # Snapshot from local Docker $ containerized-strands-agents snapshot --data-dir ./researcher --output snap.zip # Restore to GitHub Actions $ gh workflow run agent.yml -f snapshot=snap.zip -f message="continue" # Pull results back to local $ containerized-strands-agents pull --repo owner/repo --run-id 12345
